# Terra -- Product Requirements Specification (PRS) v1.1 Mobile

## 1. Product Vision

Terra is a mobile-first ecological intelligence platform delivering
monthly satellite-based structural insights and mission-driven
restoration guidance.

## 2. Platform Strategy

Phase 1 will be developed using React Native to support iOS and Android
from a shared codebase. Platform-specific modules may be implemented for
offline storage, push notifications, and performance optimisation.

## 3. Core Functional Requirements (Phase 1)

• Plot boundary interface

• Monthly ecological composite automation

• Regional ecosystem identification

• Native vegetation preference engine

• Mission recommendation system

• Push notification engine

• Offline-first capability for briefs and missions

## 4. Non-Functional Requirements

• Monthly processing cadence

• Secure local data storage

• Background sync architecture

• Scalable cloud-native backend

• Clear Tier transparency (Tier 1 vs Tier 2)

• Performance optimisation for map rendering

## 5. Future Requirements (Phase 2+)

• Acoustic sensor integration

• Portfolio HHI scoring

• Elite eligibility tracking

• External validation workflow integration
